下載完安裝MySQL後，我們將MySQL Workbench打開，它主要就是資料庫的介面。打開介面會發現有一個預設好的資料庫，大致上如下，密碼則是安裝過程中設立的密碼:

回到我們的Node當中，一般來說會建立一個utils的資料夾，專門放置一些特定用途的檔案，建立資料庫連結就是其中一種。在Node中先安裝mysql2的套件npm install mysql2，它主要將MySQL的各種連接方式整合在一起，最後我們只需要建立一個database.js的檔案，就可以順利完成連接。

// In database.js
const mysql2 = require('mysql2');

// createPool()可以有多個同步連接，createConnection()只有一個連接
const pool = mysql2.createPool({
  // 主機名稱
  host: 'localhost',
  // 資料庫使用者名稱
  user: 'root',
  // 資料庫名稱
  database: 'table1',
  // 登入資料庫的密碼
  password: 'mypassword'
  // 資料庫的埠
  port: 3306
})

// 資料庫的回應一定是非同步，因此匯出Promise()來處理
module.exports = pool.promise();

原則上連接MySQL的方式有很多種，在這裡我使用createPool的方式建立連接，它主要可以建立Node和資料庫的通道。建立完成後，匯入需要使用SQL語法的檔案中，就可以和資料庫互動。

舉例來說，我們在app.js檔案中使用:

const db = require('./utils/database);

db.execute(SELECT * FROM table1)
  .then(result => {
    console.log(result);
  })
  .catch(err => {
    console.log(err)
  });

基本上將SQL語法搭配Express框架，我們就能夠去處理一個完整網站的後端回應。今天單純地講過使用MySQL最基礎的連線方式，基本上連線成功後就能夠在Node當中寫各種SQL語法，不僅可以透過Template Engine處理網站頁面，還能夠透過SQL處理各式各樣的資料，簡直就是全端!! 不過對於不熟悉SQL的前端工程師來說，要去撰寫SQL語法實在是有點痛苦，因此Sequelize就出現來幫助我們處理各種複雜的SQL語法，接下來我們只要會踩離合器和油門就能夠在路上奔馳了!

MVC(Model-View-Controller)架構

Do not waste your time in anger, regrets, worries, and grudges.

MVC是軟體架構的一種概念，我們將軟體開發以Model(模型)、View(介面)、Controller(控制器)作歸類。套用MVC可以避免程式開發所有的邏輯相互混雜，影響後續的維護和擴充，事前的縝密規劃需要時間，但換來的報酬想必會相當值得，特別是在開發大型的應用程式。完善的MVC架構可以再進一步導入持續整合(Continuous Integration)和持續部署(Continuous Deployment)到團隊當中，在開發軟體上相當有幫助。

今天會透過Express框架以MVC架構作一個簡單的實作，藉由實作理解MVC在程式中如何發揮作用。

–預備知識–

  • Javascript: 回呼函數、類別定義
  • Express: Route
  • HTML: Template Engine

–文章大綱–

何謂MVC

Model(模型): 負責程式碼邏輯定義與判斷，主要的任務是操作資料並且與資料庫互動。
View(介面): 前端工程師/設計師規劃介面的樣式。
Controller(控制器): 作為Model與View的橋樑，一般指的是處理路由請求與回應。

實作MVC

首先建立一個新的專案，在專案內新增app.js的檔案後，依序執行以下指令:
npm init
npm install --save-dev nodemon
npm install --save express
npm install --save pug
安裝完成後，在package.json檔案的script部分新增"start": "nodemon app.js"完成第一步驟，接下來我們可以專注在我們的程式碼了!

在專案中分別建立routes、models、views、controllers四個資料夾後將app.js改成下方的程式碼:

// In app.js
const express = require('express');
const path = require('path');

const app = express();

// 匯入userRoutes的路由
const userRoutes = require('./routes/user');

// 設置要使用的樣板引擎
app.set('view engine', 'pug');
// 樣板的位置: 預設是當前路徑 + /views
app.set('views', 'views');

// 解析request的body
app.use(express.urlencoded({ extended: false}));

app.use(userRoutes);

app.listen(3000);

app.js的用途主要是設置的Express組態，像是樣板引擎和要使用的CSS的資料夾路徑，詳細可以參考Express系列文章。完成最基本的設置後，我們即將進入MVC架構。

我們將依序MVC順序開始整個架構，今天會製作簡單表單新增User及連結轉跳頁面瀏覽資料。
首先是Model，建立一個user.js的檔案，如下:

const path = require('path');
const fs = require('fs');

// 輸出檔案的路徑
// data: 資料夾名稱 
// users.json: 檔案名稱
// require.main.filename: 執行此程式碼檔案的位置 + /data/users.json
const filePath = path.join(
  path.dirname(require.main.filename),
  'data',
  'users.json',
);

// 讀取users.json檔案的資料
const getUserContent = (cb) => {
  // fs.readFile(檔案路徑, 回呼函數)
  fs.readFile(filePath, (error, userContent) => {
    if (error) {
      cb([]);
    } else {
      cb(JSON.parse(userContent));
    }
  });
};

// 定義Model中使用者的資訊
module.exports = class User {
  constructor(uid, fname, lname, img) {
    this.uid = uid;
    this.fname = fname;
    this.lname = lname;
    this.img = img;
  }

  // 使用者方法
  save(){
    getUserContent(users => {
      // 隨機給一個id值
      this.uid = Math.random().toString();
      // this指向新建立的user實體
      users.push(this);
      fs.writeFile(filePath, JSON.stringify(users), (err) => {
        console.log(err);
      })
    })
  }

  // static可以在非實體情況下使用: 不用 new User()就可以直接使用 
  static fetchUser(cb){
    getUserContent(cb);
  }
};

完成Model的定義後下一步是View介面，這裡我們撰寫兩個檔案，一個是主頁Home另外一個則是顯示資料的Users的檔案。這裡樣板我使用Pug(舊稱Jade)，若有問題的話，可以翻閱一下官方文件。

    meta(http-equiv="X-UA-Compatible", content="IE=edge")
    meta(name="viewport", content="width=device-width, initial-scale=1.0")
    title Home
  body 
    form(action="/" method="POST")
      label(for="fname") Fname
      input(type="text" name="fname" id="fname")
      label(for="lname") Lname
      input(type="text" name="lname" id="lname")
      label(for="img") Photo
      input(type="text" name="img" id="img")
      button(type="submit") Add User
    a(href="/users") Users
// In users.pug
doctype html
html(lang="en")
  head
    meta(charset="UTF-8")
    meta(http-equiv="X-UA-Compatible", content="IE=edge")
    meta(name="viewport", content="width=device-width, initial-scale=1.0")
    title Users
  body 
    each user in users 
      ul 
        li= user.uid
        li= user.fname

因為沒有使用CSS會很醜，還請多多見諒，希望能將重心放在MVC本身上。完成簡單的View後要來進入最後的Controller，Controller如同前述，主要就是處理各種路由，透過清楚劃分權責，可以在執行專案上更明確。

// Controller: In user.js
匯入整個User Class可以使用static的方法
const User = require('../models/user');

exports.getHome = (req, res, next) => {
  res.render('home');
}

exports.postAddUser = (req, res, next) => {
  // Express目前將bodyParser直接設置好，不用額外引入bodyParser這個Lib
  const fname = req.body.fname;
  const lname = req.body.lname;
  const img = req.body.img;
  // 建立user實體，因為我們需要"新"的User
  const user = new User(null, fname, lname, img);
  user.save();
  res.redirect('/');
};

// 讀取檔案中的Users
exports.getUsers = (req, res, next) => {
  User.fetchUser((users) => {
    res.render('users', {
      users: users,
    });
  });
};

這裡的Controller寫的相當簡單，主要有Home的GET和POST方法的路由，最後則是User的GET方法路由。實際執行，主頁就會跑出來一個不美觀的表單和一個連結!

樣板引擎(Template Engine)

樣板引擎又稱作樣板處理器，簡單來說就是將語法和資料結合在一起的轉換器。透過樣板引擎所寫成的頁面，最後會轉換成常見的HTML檔案，因此樣板引擎也稱作前或預處理器。

今天將會分別介紹EJS、PUG和Handlebars三個不同的樣板引擎來簡單理解它們的使用方式與差異。

EJS

首先在全新的資料夾下建立app.js後執行npm init初始化檔案的設定檔，接著安裝nodemon和express以及ejs三個套件。

到目前為止都有在介紹Routes時提到，忘記的話可以回上一篇文章複習一下↑。

首先使用EJS前，必須先在Express設置EJS作為樣板引擎，並且指定樣板的位置路徑給Express。

如下:

const express = require('express');
const app = express();

// 設定樣板引擎
app.set('view engine', 'ejs');
// 樣板的路徑 -> 從app.js的路徑看起 -> 會找到和app.js同層級的views資料夾內
app.set('views', 'views');

app.listen(3000);

隨後建立views資料夾，在其中分別建立home.ejs這個檔案，並且撰寫如下:
home.ejs

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- <%= %> 用於傳遞變數給EJS樣板 -->
  <title><%= pageTitle %></title>
</head>
<body>
  <h3>This is Home</h3>
</body>
</html>

EJS的特別處在於可以透過<%= %>傳遞變數或者<% %>去執行邏輯，在撰寫上較為直覺。接著再將app.js稍作修改後，就可以看見成功地在Express使用EJS樣板引擎。
app.js

const express = require('express');
const app = express();

// 設定樣板引擎
app.set('view engine', 'ejs');
// 樣板的路徑 -> 從app.js的路徑看起 -> 會找到和app.js同層級的views資料夾內
app.set('views', 'views');

app.get('/', (req, res, next) => {
  // render函數 -> 參數1: 渲染的檔案  參數2: 傳入的物件 
  res.render('home', { pageTitle: 'Home'});
});

app.listen(3000);

執行npm start後打開瀏覽器輸入localhost:3000，看見Home正對著我們揮手!

除了輸出的This is Home文字外，網頁Title順利存取傳遞至home.ejs的物件屬性pageTitle。

PUG

現在我們來透過PUG達到一樣的結果，首先透過npm install –save pug安裝套件後，將app.js的樣板引擎更改至app.set('view engine', 'pug')，其餘的不更動。

接著我們建立home.pug的檔案在views資料夾內，如下:
home.pug

doctype html
html(lang="en")
  head
    meta(charset="UTF-8")
    meta(http-equiv="X-UA-Compatible", content="IE=edge")
    meta(name="viewport", content="width=device-width, initial-scale=1.0")

    <!-- #{} 用於傳遞變數給樣板 -->
    title #{pageTitle}
    link(rel="stylesheet", href="/css/main.css")
    body 
      h3 This is Home

PUG本身使用上會較EJS簡潔，因為模板是透過成對標籤定位，因此縮排會嚴重影響程式碼的結果，需要特別注意。接著執行來到瀏覽器後會得到和EJS一樣的結果。

handlebars

使用handlebars前一樣需要透過npm安裝，注意是要執行npm install --save express-handlebars而不是只有handlebars。handlebars的設置較EJS和PUG不同，主要是因為Express並沒有預先寫好handlebars相關的組態，因此需要多一些步驟，但並不困難，如下:
app.js

const handlebars = require('express-handlebars');

// 執行handlebars樣板函數並設置hbs變數名稱為引擎名稱 
// layoutsDir: 設定樣板的路徑 defaultLayout: 指定樣板架構 extname: 顯示副檔案名稱
app.engine('hbs', handlebars({ layoutsDir: 'views', defaultLayout: 'home', extname: 'hbs'}));
// 指定樣板為hbs
app.set('view engine', 'hbs');

handlebars在設置上比較麻煩，因為Express本身沒有處理它，但在最後都會得到一樣的結果。

最後我們一樣建立home.hbs的檔案，大致上和EJS較為相近:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  {{!-- {{ }} 用於傳遞變數給EJS樣板 --}}
  <title>{{ pageTitle }}</title>
</head>
<body>
  <h3>This is Home</h3>
</body>
</html>

結語

Express介紹

誠如先前所說，Express是伺服器框架其一 ，其餘的類似的框架有Laravel、Flask等等。Express主要的用途在於協助Node去打造Web應用程式的框架和API，大幅減輕開發的負擔。以往在Express開發前，處理API和Web應用程式，需要針對網頁的請求與回應逐一處理，但現在只需要透過Express就可以達到一樣的結果。Express扮演的角色就像是遙控器一樣，開啟電視只需要按下開機關機鍵，若沒有遙控器的話則必須直接按電視螢幕本身的開關，但結果上來看是一樣的。

簡言之，Express就是Node處理網站的遙控器!

Express使用

在使用Express前，我們需要先認識中介軟體(Middleware)，中介軟體本身可以想成火車，假設火車從台中北上開往台北，這一路上必須經過苗栗、新竹、桃園、新北幾個城市，而中介軟體則是城市與城市間的溝通方式，並且溝通結果會影響到接下來的行程。
